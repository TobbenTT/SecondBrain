# Skill: Identificar Proxima Accion (GTD)

## Proposito
Para cada proyecto activo, identificar la UNICA proxima accion fisica que mueve el proyecto hacia adelante.

## Principio GTD
"La proxima accion es la proxima actividad fisica y visible que acerca el proyecto a su completitud."
— David Allen

## Proceso

### 1. Revisar Estado del Proyecto
- ¿Que sub-tareas estan completadas?
- ¿Que sub-tareas estan pendientes?
- ¿Hay alguna bloqueada esperando a alguien? (→ Waiting For)

### 2. Determinar la Proxima Accion
La proxima accion DEBE cumplir:
- Es una accion FISICA (no "pensar en..." sino "escribir el email a...")
- Es CONCRETA (no "avanzar con el proyecto" sino "llamar a Juan para confirmar fecha")
- Es UNICA por proyecto (solo una a la vez)
- Es EJECUTABLE ahora (no depende de otra tarea pendiente)
- Tiene un RESPONSABLE claro

### 3. Clasificar la Proxima Accion
Asignar:
- **Contexto GTD**: ¿Donde se ejecuta?
  - @computador: Trabajo digital
  - @email: Enviar/responder correo
  - @telefono: Llamada telefonica
  - @oficina: Presencial en oficina
  - @reunion: Necesita reunion/coordinacion
  - @espera: Delegada, esperando respuesta
  - @calle: Diligencia fuera de oficina
  - @investigar: Requiere busqueda/analisis
- **Energia**: ¿Cuanta concentracion necesita?
  - baja: Tareas mecanicas, copiar, organizar
  - media: Coordinacion, emails complejos, revision
  - alta: Analisis, redaccion, presentaciones, decisiones
- **Tiempo**: ¿Cuanto toma? (15min, 30min, 1h, 2h, 4h, 1dia)

### 4. Patron de Actualizacion
Cuando una proxima accion se COMPLETA:
1. Marcar como completada
2. Buscar la siguiente sub-tarea pendiente
3. Marcar esa como nueva proxima accion
4. Si no hay mas sub-tareas → proyecto completado

## Anti-patrones (EVITAR)
- "Revisar proyecto" → No es accionable. ¿Que especificamente revisar?
- "Trabajar en X" → Demasiado vago. ¿Cual es el primer paso concreto?
- "Coordinar con el equipo" → ¿Quien? ¿Sobre que? ¿Por que medio?
- Multiples proximas acciones → Solo UNA por proyecto

## Ejemplo
Proyecto: "Implementar plan de capacitacion Q2"
- ~~Definir malla curricular~~ (completada)
- ~~Aprobar presupuesto con Finanzas~~ (completada)
- **→ Enviar email a proveedor de cursos solicitando cotizacion** (PROXIMA ACCION - @email, energia: media, 30min)
- Revisar propuestas recibidas (pendiente)
- Programar fechas en calendario (pendiente)
