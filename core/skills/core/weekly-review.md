# Skill: Revision Semanal GTD

## Proposito
Hacer una revision completa del sistema cada semana para mantenerlo confiable y actualizado.

## Frecuencia
Una vez por semana (recomendado: viernes por la tarde o lunes por la manana).

## Checklist de Revision Semanal

### 1. Vaciar el Inbox (Capturar → Organizar)
- [ ] Revisar TODAS las ideas en estado "captured" (inbox)
- [ ] Clasificar cada una: ¿Es accionable? ¿Es proyecto o tarea?
- [ ] Asignar contexto GTD, energia, responsable, tiempo
- [ ] No dejar NADA en el inbox sin procesar

### 2. Revisar Proximas Acciones
- [ ] ¿Cada proyecto activo tiene UNA proxima accion definida?
- [ ] ¿Las proximas acciones son realmente accionables?
- [ ] ¿Hay acciones completadas sin marcar?
- [ ] ¿Hay acciones estancadas >5 dias? → Re-evaluar o delegar

### 3. Revisar Delegaciones (Waiting For)
- [ ] ¿Que estamos esperando de quien?
- [ ] ¿Hay delegaciones vencidas? → Seguimiento
- [ ] ¿Alguna delegacion ya se completo sin registrar?

### 4. Revisar Proyectos
- [ ] ¿Cada proyecto tiene resultado definido?
- [ ] ¿Hay proyectos sin movimiento en >2 semanas? → ¿Sigue activo?
- [ ] ¿Hay proyectos para archivar?
- [ ] ¿Algun "Algun dia / Tal vez" se volvio urgente?

### 5. Revisar Areas de Responsabilidad
- [ ] ¿Alguna area tiene demasiada carga?
- [ ] ¿Alguna area esta descuidada (0 acciones)?
- [ ] ¿El balance de carga entre consultores es adecuado?

### 6. Revisar Calendario (proximas 2 semanas)
- [ ] ¿Hay fechas limite proximas?
- [ ] ¿Hay compromisos que necesitan preparacion?
- [ ] ¿Fechas que ya pasaron sin completarse?

### 7. Limpiar y Archivar
- [ ] Mover ideas inactivas >30 dias a Archivo
- [ ] Limpiar material de apoyo obsoleto
- [ ] Actualizar contexto/memoria si hubo cambios organizacionales

## Metricas de Revision
- Total ideas en inbox (deberia ser 0 al final)
- Total proximas acciones activas
- Total delegaciones pendientes
- Proyectos sin proxima accion (deberia ser 0)
- Ideas estancadas >7 dias
- Tasa de completitud semanal (completadas / total activas)

## Output Esperado
Un reporte markdown con:
1. Resumen ejecutivo (estado general del sistema)
2. Acciones urgentes (lo que DEBE hacerse esta semana)
3. Proyectos bloqueados o en riesgo
4. Delegaciones vencidas
5. Sugerencias de mejora
6. Metricas comparadas con semana anterior
